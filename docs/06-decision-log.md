| D38 | **Actions input = two modes; no "loading pack".** Considered giving actions their own swappable *Loading/Actions Pack* axis (structure-type × jurisdiction — AS 1170 / AS 5100.2 / **AS 4997 maritime** / EN 1991) parallel to the geotechnical Code Pack, to catalogue & combine special-structure actions (berthing, mooring, wave…). **Rejected as over-complex** and unfriendly to a geotech engineer unfamiliar with special actions (founder). **Adopted instead:** the **action combinations belong to the structural/loading code, not the geotech tool.** **Mode A (default):** the engineer enters the structural engineer's **already-combined design action effects** (ULS `V_d`/`H_d`/`M_d` + SLS) directly, consumed as-is — handles *any* loading code / special structure transparently, and matches the real division of labour + flexible-entry (D15). **Mode B (fallback):** enter **characteristic** actions and let the tool combine — for simple cases and, crucially, **EC DA1** (which needs A1 & A2 action sets, so direct entry alone is ambiguous). Captured in the stage doc "Actions & load combinations" section, procedure 5.0.4 (two modes), and Method Library §5.x(EC). | Accepted |
